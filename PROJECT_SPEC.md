# プロジェクト仕様書: library-calendar

## 1. プロジェクト概要
「library-calendar」は、複数の図書館の開館・休館状況を一覧表示、および管理するためのWebアプリケーションです。各図書館の複雑な休館ルール（定期休館、祝日振替、特別休館など）を自動判定し、ユーザーが今日や指定した日付の状況を簡単に確認できる機能を提供します。

## 2. 技術スタック
- **開発ツール**: Vite
- **言語**: HTML, Vanilla JavaScript (ES Modules), CSS
- **PWA**: Web App Manifest, Service Worker
- **デプロイ**: GitHub Actions (GitHub Pages)

## 3. ディレクトリ構成
- `index.html`, `app.js`, `style.css`: 閲覧画面（メイン）
- `edit.html`, `edit.js`, `edit.css`: カレンダー編集画面
- `data.json`: 図書館情報、祝日データ、年度別特別ルールを格納するデータファイル
- `public/`: マニフェストファイル、アイコン、Service Workerなどを格納
- `.github/workflows/deploy.yml`: 自動デプロイ設定

## 4. 画面仕様

### 4.1 閲覧画面 (`index.html`)
- **日付選択**: 指定した日付の状況を確認可能。
- **クイックリンク**: 「今日」「明日」「明後日」の状況をワンタップで切り替え。
- **図書館カード**:
    - 図書館名、開館時間（または休館表示）、現在のステータス（開館中、開館前、閉館、休館日）を表示。
    - カードをタップすると各図書館の公式サイトへ遷移。

### 4.2 編集画面 (`edit.html`)
- **ライブラリ選択**: 編集対象の図書館を切り替え。
- **カレンダー表示**: 月単位の開館状況を可視化。
- **個別編集**: カレンダーの日付セルをクリックすることで、その日の「特別開館」または「特別休館」をトグル形式で切り替え可能。
- **JSONエクスポート**: 編集後のデータをJSON形式でクリップボードにコピーし、`data.json` を更新するための機能。

## 5. データ構造 (`data.json`)
データは大きく以下の3つのセクションで構成されます。

- **`libraries`**: 図書館の基本情報（ID、名前、URL、開館時間、基本ルール）。
    - `rules`: `weekly_closed`（曜日指定）, `holiday_open`（祝日開館の可否）, `new_year_closed`（年末年始）など。
- **`holidays`**: 祝日の配列（YYYY-MM-DD形式）。
- **`FY2025`, `FY2026`**: 年度別の特別ルール。
    - `special_closed`: 特定の休館日。
    - `special_open`: 基本ルールでは休館だが特別に開館する日。
    - `third_tues`, `third_wed`: 第3火曜・水曜などの特定ルール用（年度による変動に対応）。

## 6. 開館・休館判定ロジック
`app.js` および `edit.js` で実装されている主要な判定フローは以下の通りです。

1.  **年末年始**: `new_year_closed` に基づく判定。
2.  **年度別特別ルール**: `special_open` / `special_closed` などの個別指定を最優先。
3.  **祝日振替ルール (`holiday_shift`)**:
    - 月曜が祝日の場合、月曜を開館し翌火曜を休館とするなどのシフト処理。
4.  **特定曜日・第n曜日**: 「毎週月曜」「第4木曜」などの定期休館。
5.  **祝日の翌日休館**: 福井県立図書館などで採用されている「祝日の翌日（平日のみ）を休館」とするロジック。

## 7. PWA機能
- **オフライン対応**: `sw.js` によりリソースをキャッシュし、オフライン環境下でも閲覧・編集機能が動作します。
- **ホーム画面追加**: `manifest.webmanifest` により、モバイル端末でアプリのようにインストール可能です。

## 8. デプロイフロー
GitHubの `main` ブランチへプッシュされると、GitHub Actions が起動し、Viteによるビルド後、自動的に `dist/` ディレクトリの内容を GitHub Pages にデプロイします。
